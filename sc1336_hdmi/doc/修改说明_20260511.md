# sc1336_hdmi 修改说明 (2026-05-11)

## 概述

本次基于参考工程对 `sc1336_hdmi` 项目进行了全面的架构调整和 bug 修复，涉及 DDR3 配置适配、AXI4 读写链路贯通、时钟复位系统完善、时序约束，以及测试验证框架搭建。

---

## 1. DDR3 配置适配

### 1.1 MIG IP 配置 (MT41K128M16XX-15E)

| 参数 | 旧值 | 新值 | 原因 |
|------|------|------|------|
| `sys_clk_i` | 200MHz | **250MHz** | 匹配新 PLL 输出 |
| `clk_ref_i` | 未连接 | **200MHz** | IDELAYCTRL 参考钟(必须 200MHz±10MHz) |
| `ddr3_addr` | `[14:0]` | `[13:0]` | MT41K128M16 行地址 14bit，MIG 输出 14bit |
| DDR 频率 | 400MHz | **666.67MHz (1333MT/s)** | 芯片最高频率 |
| PHY 比例 | — | **4:1** | UI Clock = 166.67MHz |

### 1.2 时钟架构

```
clk_50m (板上晶振)
  ├─ sys_clk_ctrl_cmos → clk_ctrl(100M) + clk_cmos(24M) + spare(50M)
  ├─ sys_clk_video     → clk_video(74.25M)
  └─ sys_clk_ddr_ref   → clk_ddr_ref(250M) + clk_ddr_ref_200m(200M)
                             │                        │
                             v                        v
                       MIG sys_clk_i             MIG clk_ref_i
```

### 1.3 ddr3_addr 位宽差异分析

参考工程用 MT41K256M16 (4Gb，行地址 15bit → `ddr3_addr[14:0]`)，本工程用 MT41K128M16 (2Gb，行地址 14bit → `ddr3_addr[13:0]`)。容量翻倍通过行数翻倍实现，与 DDR 型号直接相关。

## 2. AXI4 信号匹配 (DDR3_0 ↔ axi4_ctrl)

### 2.1 位宽对齐

| 信号 | axi4_ctrl 输出 | MIG 期望 | 处理 |
|------|---------------|----------|------|
| `awid/arid` | `[7:0]` | `[3:0]` | 高位截断 |
| `awaddr/araddr` | `[31:0]` | `[28:0]` | 高位截断 |
| `awlock/arlock` | `[1:0]` | `[0:0]` | 仅取 LSB |
| `awregion/arregion` | `[3:0]` | 无 | 悬空 |

### 2.2 新增 40 条 AXI 连线

之前 AXI 通道信号 (wdata, wstrb, wlast, bvalid, arid, rdata 等) 多数悬空，补齐了全部 40 条连线的声明和连接。

## 3. axi4_ctrl 写状态机 Bug 修复

### 3.1 状态机 case 标签错位 (致命)

状态编码：
```
WS_W_IDLE  = 2'b00
WS_W_WDATA = 2'b01
WS_W_WINC  = 2'b10
WS_W_EOF   = 2'b11
```

原 case 语句中 `WS_W_EOF` 标签出现了两次，导致 `WS_W_WINC`(2'b10) 无 case 匹配，写完一个 burst 后死锁。同时 EOF 处理逻辑错误(依赖永远不为 1 的 `rc_w_eof` 信号)，帧结束时死锁。

**修复**：重写 case 分支，每个状态编码对应唯一的 case 标签和正确的处理逻辑。

### 3.2 awvalid 不清零 (严重)

```verilog
// 改前 (无条件 hold)
end begin  axi4_awvalid <= axi4_awvalid;  end

// 改后 (awready 握手后清零)
if(axi4_awvalid && axi4_awready)  axi4_awvalid <= 1'b0;
```

awvalid 永远为 1，MIG 收不到新的 AXI 写地址请求，只有第一个 burst 可能被接受。

### 3.3 异步复位敏感列表错误

```verilog
// 改前
always @(posedge axi4_clk or posedge axi4_rst_n)

// 改后
always @(posedge axi4_clk or negedge axi4_rst_n)
```

axi4_rst_n 是低有效复位，标准写法应该用 `negedge`。

### 3.4 多驱动移除

`r_wframe_inc` 和 `r_rframe_inc` 在两个不同的 always 块中被赋值，违反了 Verilog 单驱动原则。移除帧索引管理块中的重复赋值，保留写/读状态机各自驱动。

### 3.5 awsize 修正

```verilog
// 改前
parameter DATA_SIZE = 4;  // 2^4=16B/beat (错误)

// 改后
parameter DATA_SIZE = 5;  // 2^5=32B/beat (匹配 256-bit 数据总线)
```

## 4. W_WIDTH 适配

`axi4_ctrl` 的 W_WIDTH 从 32 改为 8，匹配 SC1336 传感器的 8-bit RAW 输出。模块内部 shift register 自动处理 256-bit FIFO 字的打包/拆包。

---

## 5. 复位系统完善

### 5.1 MIG sys_rst 生成

`sys_rst_n`(低有效) 不能直接接 MIG `sys_rst`(高有效)，在 `sys_clk_ctrl` 内新增异步置位/同步释放电路：
- 异步置位: `sys_rst_n=0` 时立即拉高 MIG 复位
- 同步释放: `clk_ddr_ref` 域 2 级同步后释放

### 5.2 Sensor_Image_XYCrop 复位

修复 `cmos_de` 未驱动问题：
```verilog
assign cmos_de = 1'b1;  // DVP: HREF 高时每 PCLK 一有效像素
```

### 5.3 cmos_frame_Gray 位宽

从 `[63:0]`(参考工程 MIPI 位宽) 改为 `[7:0]`(SC1336 DVP 8-bit)。

## 6. LCD 测试框架

### 6.1 lcd_display_test

从 `hdmi_display` 工程移植测试图案生成器，4 种图案 0.5s 循环切换(水平彩条/垂直彩条/分形/灰度渐变)。

```verilog
wire TEST_MODE = 1'b1;  // 1=测试图案, 0=DDR 数据
```

### 6.2 DDR 测试图案生成器

使用 BUFGMUX 切换写时钟(内部 `clk_cmos`/外部 `cmos_pclk`)，生成 1280×720 XOR 灰度测试帧，带完整消隐区：

```verilog
wire TEST_DDR = 1'b1;  // 1=DDR 测试, 0=传感器
```

**水平消隐**: 120 PCLK / 行  
**垂直消隐**: 10 行 / 帧

## 7. 时序约束 (timing.xdc)

### 7.1 时钟定义

| 时钟 | 周期 | 来源 |
|------|------|------|
| clk_50m | 20ns | 板上晶振 |
| CMOS_PCLK | 41.667ns | 传感器输入 |
| UI_CLK | 6ns | MIG 输出 (MIG in_context.xdc 自动创建) |

### 7.2 异步时钟组

```tcl
Group A: clk_50m + 所有 MMCM 衍生时钟 (同源, Vivado 自动分析)
Group B: CMOS_PCLK (传感器独立晶振)
Group C: MIG UI_CLK (OOC 打断推导链)
```

三组互异步，跨域通过 W0_FIFO_256 / R0_FIFO_256 保证。

### 7.3 I/O 约束

- DVP 输入: `set_input_delay -clock CMOS_PCLK -max 8ns -min 2ns`
- 时钟输出 (`cmos_xclk`, `lcd_dclk`): 不加 output delay (它们本身就是时序基准)
